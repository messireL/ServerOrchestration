# ServerOrchestration v0.1.53

## Что сделано
- синхронизированы version-маркеры до `0.1.53` в `VERSION`, backend runtime, frontend и локальном Docker build-контуре;
- обновлены `app/backend/Dockerfile` и `deploy/docker-compose.local.yml`, чтобы версия не отставала от релиза;
- из архива исключён `.pytest_cache`;
- добавлен `.gitignore` для `.pytest_cache`, `__pycache__` и `*.pyc`;
- перевыпущена документация релиза и transfer file.

## Почему появился этот релиз
`v0.1.52` по функционалу подписки был полезным, но упаковка была грязной: в архив попал тестовый кэш, а часть version-маркеров осталась не синхронизирована. Такой хвост лучше не замазывать словами, а закрывать отдельным чистым фикс-релизом.

## Что проверить
1. В UI и `/health` версия = `0.1.53`.
2. В архиве нет `.pytest_cache`.
3. Отображение `Подписка 3x-ui` осталось нормальным и русским.
4. Docker/local build-контур не тащит старый version marker.

## Следующий шаг
- проверить живой deploy `v0.1.53`;
- затем вернуться к разгрузке экрана диагностики и дальнейшему functional monitoring release.
