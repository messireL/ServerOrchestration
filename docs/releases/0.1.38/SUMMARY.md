# ServerOrchestration v0.1.38

## Что вошло
- исправлен 422 на `POST /api/probes/ssl/run`;
- исправлен такой же сценарий для ручного запуска 3x-ui;
- backend теперь принимает пустой POST и подставляет дефолтные timeout;
- frontend отправляет дефолтные payload для SSL/3x-ui;
- сообщения API-ошибок больше не разваливаются в `[object Object]`.

## Проверка
1. Нажать `Проверить SSL`.
2. Нажать `Проверить 3x-ui`.
3. Убедиться, что нет `422 Unprocessable Entity`.
4. Если ошибка есть, в UI должен отображаться человекочитаемый текст.
