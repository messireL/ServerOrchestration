# Release 0.1.19

## Что сделано
- backend image подготовлен для non-root ICMP ping: добавлен `setcap cap_net_raw=+ep` для системного `ping`;
- в compose для backend добавлен `cap_add: NET_RAW`;
- добавлен endpoint `/api/probes/ping/diagnostics` для проверки наличия `ping` и self-test внутри контейнера;
- UI при failed ping теперь показывает warning с первой причиной ошибки, а не только сухое `failed=1` без намёка, где именно всё умерло;
- документация дополнена пояснением по Portainer: почему при redeploy контейнеры могут выглядеть новыми и откуда берутся stopped-хвосты.

## Почему это понадобилось
После `0.1.18` endpoint перестал падать `500`, но на стенде ping всё ещё не проходил. Значит проблема была уже не в таблице и не в latency, а в реальном выполнении ICMP внутри non-root контейнера.
