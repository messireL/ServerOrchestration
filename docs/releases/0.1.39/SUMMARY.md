# ServerOrchestration v0.1.39

## Что вошло
- ручной и фоновый запуск проверки 3x-ui больше не ломается на `500 Internal Server Error` из-за несовместимого вызова `update_3xui_status`;
- для проверки 3x-ui console отключена жёсткая TLS-валидация, поэтому self-signed панели теперь не отваливаются только из-за сертификата;
- в summary/status теперь сохраняются расширенные детали 3x-ui subscription: число конфигов, типы ссылок, профиль, кодировка, `subscription-userinfo`, коды ответа и время ответа;
- в summary/status теперь сохраняются расширенные детали SSL: CN/issuer, self-signed, остаток дней, признак hostname mismatch и связанный checked_at;
- карточки мониторинга на фронте теперь показывают более полезные короткие детали по 3x-ui и SSL вместо полупустых статусов.

## Проверка
1. Нажать `Проверить 3x-ui` для сервера с self-signed 3x-ui.
2. Убедиться, что ручной запуск не падает в `500`.
3. Проверить, что в карточке сервера у 3x-ui/subscription появились детали уровня `HTTP 200 · N cfg · ...`.
4. Нажать `Проверить SSL`.
5. Проверить, что в карточке сервера у SSL появились CN / self-signed / остаток дней либо нормальная ошибка.
6. Открыть список серверов после автообновления и убедиться, что те же детали сохраняются после фонового scheduler-прогона.
