# ServerOrchestration v0.1.51

Дата подготовки: 2026-04-15

## Что сделано
- усилен backend-парсер HTML-страниц подписок 3x-ui, чтобы он не подхватывал заголовки страницы и куски HTML/JS как `profile_title`;
- добавлен отдельный проход по текстовым линиям страницы подписки, чтобы вытаскивать пары `label -> value` даже без нормальной таблицы;
- улучшен script/JSON fallback-парсер для случаев, когда 3x-ui кладёт `download/upload/total/expire/subId/lastOnline` в JS-объект страницы с некавычёнными ключами;
- добавлена фильтрация подозрительных значений, похожих на HTML-разметку, inline-script и технический мусор;
- добавлены backend unit-тесты на три сценария: HTML-таблица, JS-объект страницы и защита от HTML-фрагментов в профиле.

## Зачем
Проблема текущего шага: вместо данных подписки UI показывал только кусок страницы/разметки. На практике это происходило из-за ложного захвата заголовка страницы и слишком слабого script fallback.

## Что проверить после обновления
1. Открыть мониторинг сервера с 3x-ui и нажать проверку подписки.
2. Убедиться, что в деталях появляются:
   - статус;
   - download/upload/used/total/remain;
   - срок действия;
   - last online / subscription id, если они реально есть на странице.
3. Проверить, что вместо куска HTML больше не показывается мусорный хвост страницы.
4. Если у конкретной подписки данные всё ещё пустые, сохранить HTML-ответ страницы подписки целиком — следующим шагом добьём конкретный шаблон 3x-ui без гадания.
