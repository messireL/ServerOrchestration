# ServerOrchestration v0.1.52

## Что сделано
- дочищено frontend-отображение subscription profile 3x-ui;
- строка подписки переведена на русский и перестала показывать техмусор вроде `HTTP 200`, `html embedded`, `cfg`, обрывков HTML/JS, если профиль уже успешно распарсен;
- в UI оставлены только понятные поля: трафик, остаток, срок, статус, last seen, subscription id;
- частично русифицированы соседние плашки/подписи мониторингового контура;
- синхронно обновлена проектная документация: README, ТЗ, архитектура, transfer file.

## Зачем
Backend уже начал вытаскивать реальные данные подписки, но frontend всё ещё примешивал к ним служебную диагностическую шелуху. Из-за этого данные были вроде бы «есть», но по факту читать их было неприятно и бессмысленно. В этом релизе UI перестаёт быть радиооператором из 2007 года и начинает говорить нормально.

## Что проверить
1. В карточке/таблице сервера строка `Подписка 3x-ui` показывает понятный русский summary.
2. Если профиль подписки найден, рядом нет хвостов `HTTP 200`, `html embedded`, `cfg` и подобных служебных меток.
3. Кнопки ручных проверок не сломались.
4. Версия в UI и `/health` = `0.1.52`.

## Следующий шаг
- проверить живые проблемные страницы подписок;
- затем вернуться к разгрузке экрана диагностики и следующему functional monitoring release.
