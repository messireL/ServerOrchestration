# Release Notes — v0.1.42

## Что исправлено
- проверка 3x-ui подписки теперь умеет разбирать не только raw/plain и raw/base64, но и HTML-страницы подписки;
- добавлен поиск встроенных share-ссылок (`vmess://`, `vless://`, `trojan://`, `ss://` и др.) прямо внутри HTML;
- добавлен поиск встроенного base64-блока внутри HTML и его декодирование;
- если endpoint отдаёт HTML-страницу, но в HTTP headers уже есть `Subscription-Userinfo`, мониторинг больше не валится в FAIL и использует эти данные для срока/лимитов/трафика.

## Что проверить после обновления
1. Нажать `Проверить 3x-ui`.
2. Для subscription badge должно появиться не только `OK`, но и hover с данными: cfg/used/total/expire, если они реально есть.
3. В ошибках не должно остаться `subscription endpoint returned HTML instead of subscription data` для рабочей ссылки, которая открывает HTML-страницу подписки.
