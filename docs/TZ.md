# Техническое задание проекта (актуальная проектная версия)

## Назначение
Система мониторинга — web-панель для управления и контроля Ubuntu-серверов с единым GUI, inventory, status-слоем, alerting, probe-проверками и будущим исполнительным контуром для Ansible.

## Обязательные требования
- основной deploy/update через GitHub + Portainer;
- Docker Compose как обязательный формат поставки;
- контейнеры приложения работают non-root;
- inventory серверов и групп хранится в PostgreSQL;
- состояние проверок хранится отдельно от логов job-ов;
- ping и лёгкие probe-проверки выполняются отдельным слоем, не через Ansible;
- Ansible используется как слой выполнения изменений и административных операций по SSH;
- версия каждого релиза обязательна;
- каждый релиз сопровождается папкой `docs/releases/<version>/`.

## UI-требования
- панель должна иметь левое меню;
- функционал должен быть разнесён по отдельным разделам;
- dashboard не должен смешивать все действия и все таблицы на одном экране;
- inventory должен поддерживать создание, редактирование и удаление серверов и групп;
- связи сервер ↔ группа должны быть обозримыми и управляемыми через GUI;
- интерфейс должен поддерживать светлую и тёмную тему.

## Актуализировано для v0.1.16
- сервер inventory поддерживает поля `host`, `ssh_port`, `ssh_user`, `web_url` и флаги мониторинга;
- `web_url` используется для проверки HTTP/HTTPS доступности;
- HTTP/HTTPS probe работает в browser-like режиме и не равен будущей проверке 3x-ui subscription;
- должны поддерживаться проверки: ping, SSH-порт, HTTP/HTTPS, затем 3x-ui, SSL, timezone, apt, reboot, журналы;
- alerts по недоступности должны создаваться отдельно по типам `ping_down`, `ssh_down`, `http_down`.

## Ближайшие функциональные блоки
- 3x-ui console checks;
- 3x-ui subscription checks с browser-mode, base64 и метаданными;
- SSL checks;
- timezone checks;
- apt update/upgrade/dist-upgrade через Ansible;
- reboot и maintenance windows;
- UFW/SSH logs;
- уведомления email и Telegram.
