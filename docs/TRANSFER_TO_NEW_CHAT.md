# ServerOrchestration — перенос в новый чат

## Что это
Веб-панель оркестрации и мониторинга серверов. Текущий подтверждённый рабочий фокус: функциональные релизы мониторинга и operational-контура, а не декоративная полировка UI.

## Текущий статус
- Текущий подготовленный релиз: **v0.1.32**
- База для продолжения: архив `ServerOrchestration-v0.1.32`
- Последний пользовательский фокус: меньше полировки интерфейса, больше функционала мониторинга и operational-логики.

## Что важно помнить
- У пользователя уже зафиксирован отдельный сценарий: **3x-ui и SSL проверяются по подписочной ссылке и её порту**, который может отличаться от SSH-порта.
- Для этого использовать `subscription_3xui_url`; не сводить проверки 3x-ui/SSL к SSH host:port.
- Кнопка «Добавить сервер» должна жить на том же стабильном AJAX-сценарии, без лишних ломких JS-наворотов.
- В тёмной теме hover по строке сервера должен оставаться читаемым.

## Что вошло в v0.1.32
- Устранён критичный backend-crash на старте: удалён импорт несуществующего `run_xui_check`.
- Довосстановлены импорты в probe-контуре (`urlparse`, `datetime/timezone`, `Any`, `Dict`).
- Сохранён сценарий проверки 3x-ui и SSL по `subscription_3xui_url` и её порту.
- Синхронизированы версионные файлы и документация релиза.

## Что проверить после выкладки
1. Добавление сервера через UI.
2. Ручную проверку 3x-ui по `subscription_3xui_url`.
3. Ручную SSL-проверку по порту ссылки подписки.
4. Ручной запуск connectivity / HTTP / 3x-ui / SSL без падения backend.
5. Отсутствие `run_xui_check` / `Dict is not defined` в логах backend.

## Следующий логичный шаг
После подтверждения v0.1.32 идти дальше в функциональный контур мониторинга: журналы probe-run, более явная операционная диагностика по типам проверок, и дальнейшее разведение разделов/дашбордов по смыслу без избыточной UI-магии.
